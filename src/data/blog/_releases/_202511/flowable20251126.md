CommandExecutor
它的作用：以统一、可控、可扩展的方式执行“命令”。这个命令可以是任何对引擎的操作。

核心思想：命令模式

flowable将所有的操作，都抽象成了Command接口，然后通过CommandExecutor来执行这些命令。 

比如：起一个流程、完成一个任务、保存一个用户或组，都是Command接口的实现。 

这种设计，把请求的发起者和请求的执行者分离开来。  

工作机制：责任链模式

CommandExecutor自身并不执行命令，它的真正实现（CommandExecutorImpl）是责任链的起点。

当你调用 commandExecutor.execute(command) 时，这个 command 对象会经过一个由多个拦截器 (CommandInterceptor) 组成的链条。


这个链条就像一个工厂的流水线，每个拦截器都是一个工位，负责一项特定的“横切关注点”：

1.LogInterceptor (日志拦截器): 记录命令的开始和结束，方便调试。

2.CommandContextInterceptor (上下文拦截器): 管理命令执行的上下文，比如事务管理、变量管理等。

3.TransactionInterceptor (事务拦截器): 负责开启、提交或回滚数据库事务。它会捕获后续步骤中的异常，并决定是 commit 还是 rollback。

4.CommandExecutorInterceptor (执行器拦截器): 负责实际执行命令。
